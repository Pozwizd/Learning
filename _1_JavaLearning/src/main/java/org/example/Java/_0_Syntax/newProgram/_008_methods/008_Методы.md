# 008. Варианты использования методов

## Перегрузка методов

Java позволяет создавать методы с одинаковым именем, если у них разный набор параметров.

```java
int sum(int a, int b) {
    return a + b;
}

double sum(double a, double b) {
    return a + b;
}
```

Это называется перегрузкой.

## Передача аргументов

Java передает аргументы по значению:

- у примитивов копируется само значение
- у ссылочных типов копируется ссылка

Из-за этого метод может менять содержимое массива, но не может простой переназначенной ссылкой заменить исходную переменную у вызывающего кода.

## Рекурсия

Метод может вызывать сам себя.

```java
int factorial(int n) {
    if (n <= 1) {
        return 1;
    }
    return n * factorial(n - 1);
}
```

Важно иметь базовый случай, иначе будет переполнение стека.

## Когда разбивать код на методы

- когда логика повторяется
- когда один блок кода стал слишком длинным
- когда части алгоритма можно дать понятные имена

## Техническая часть

- перегрузка зависит от параметров, а не только от возвращаемого типа
- рекурсия удобна не всегда: иногда цикл проще и безопаснее
- методы должны упрощать код, а не дробить его бессмысленно

## Типичные ошибки

- перегружать методы слишком похожими сигнатурами
- использовать рекурсию там, где проще цикл
- создавать методы с неясными названиями

## Практика

1. Перегрузите метод для `int` и `double`.
2. Напишите рекурсивный факториал.
3. Сравните рекурсивное и циклическое решение одной задачи.
