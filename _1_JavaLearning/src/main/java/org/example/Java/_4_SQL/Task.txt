## Работа через MySQL Workbench
Реализовать SQL БД магазина, в которой учесть:
1. Необходимо хранить в таблице Users основную информацию о пользователе.
2. В таблице User Details хранить дополнительную информацию о пользователе. Для каждого Users должен быть один User Details.
3. Также в БД должна быть информация о продуктах в таблице Products.
4. У каждого пользователя должна быть корзина для продуктов Shopping Cart.
5. Каждый пользователь может иметь много продуктов в корзине и каждый продукт может быть одновременно у многих пользователей.
6. Необходимо хранить информацию о заказах в таблице Orders. У каждого пользователя может быть много заказов, но один заказ может быть только у одного пользователя. В таблице Orders, как минимум, необходимо хранить информацию о заказанных продуктах(перечень названий продуктов через “,” в виде текста), а также общую сумму этих продуктов.
7. Какие именно данных должны быть в этих таблицах подобрать самостоятельно

- Написать SQL запросы которые используют join(inner, left, right), subquery, correlated subquery, aggregation functions.
- Примеры запросов - клиент который купил больше всего товаров, тoп 10 самых дорогих товаров, товары с ценой выше средней
- В качестве БД используем MySQL
