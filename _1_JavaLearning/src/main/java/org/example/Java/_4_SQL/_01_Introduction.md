# 1. Введение в SQL (Introduction to SQL)

## Что такое SQL?
**SQL (Structured Query Language)** — это стандартный язык программирования для управления реляционными базами данных (RDBMS) и манипулирования данными в них.

## Реляционные базы данных (RDBMS)
RDBMS (Relational Database Management System) хранит данные в структурированном формате, используя строки и столбцы.
*   **Таблица (Table)**: Основной объект хранения данных.
*   **Строка (Row/Record)**: Отдельная запись данных.
*   **Столбец (Column/Field)**: Атрибут данных определенного типа.

**Примеры RDBMS**:
*   PostgreSQL
*   MySQL
*   Oracle Database
*   Microsoft SQL Server
*   SQLite

## RDBMS vs NoSQL
| Характеристика | RDBMS (SQL) | NoSQL |
| :--- | :--- | :--- |
| **Структура** | Табличная, жесткая схема | Документы, Графы, Key-Value, Wide-Column |
| **Масштабируемость** | Вертикальная (мощнее сервер) | Горизонтальная (больше серверов) |
| **Связи** | Сложные JOIN-запросы | Обычно денормализованные данные |
| **Примеры** | PostgreSQL, MySQL | MongoDB, Redis, Cassandra, Neo4j |

## ACID: Свойства транзакций
Для обеспечения надежности данных RDBMS следуют принципам ACID:
1.  **Atomicity (Атомарность)**: Транзакция выполняется целиком или не выполняется вовсе.
2.  **Consistency (Согласованность)**: Транзакция переводит БД из одного согласованного состояния в другое.
3.  **Isolation (Изоляция)**: Параллельные транзакции не влияют друг на друга (в идеале).
4.  **Durability (Долговечность)**: После фиксации (COMMIT) данные сохранены навсегда, даже при сбое питания.

## CAP Теорема (для распределенных систем)
В распределенной системе невозможно одновременно гарантировать все три свойства:
1.  **Consistency (Согласованность)**: Все узлы видят одни и те же данные одновременно.
2.  **Availability (Доступность)**: Каждый запрос получает ответ (успех/неудача).
3.  **Partition Tolerance (Устойчивость к разделению)**: Система работает даже при потере связи между узлами.

*   **SQL (RDBMS)** обычно выбирают **CA** (или CP в кластерах).
*   **NoSQL** часто выбирают **AP** или **CP**.

## История SQL
*   **1970**: Эдгар Кодд (IBM) публикует статью о реляционной модели.
*   **1974**: Разработан язык SEQUEL (Structured English Query Language), позже переименован в SQL.
*   **1986**: SQL становится стандартом ANSI.
*   **1999**: SQL:1999 (добавлены триггеры, рекурсивные запросы).
*   **2011+**: Поддержка JSON, XML, временных таблиц.
