# Structured Concurrency и Virtual Threads в Java

## 1. Задача: "Обработка исключений для системы заказов"

### Условие

Использовать `StructuredTaskScope` для запуска всех трех задач параллельно, дождаться их завершения и вывести результат.

Представьте, что вы делаете сервис для одного клиента, который одновременно выполняет три задачи:

- Получение заказов из базы (имитация через `Thread.sleep`)
- Проверка статуса доставки (имитация)
- Подсчет общей стоимости заказов (имитация)

## 2. Задача: "Обработка 1000 клиентов с блокирующим I/O"

### Условие

Представьте, что у вас есть сервис, который обрабатывает 1000 клиентов одновременно.

Каждый клиент делает блокирующий запрос к базе данных (например, получение списка заказов) и блокирующий запрос к внешнему API (например, проверка статуса доставки).

Для целей задания можно использовать заглушки, которые имитируют задержку через `Thread.sleep()` вместо реальных сетевых или БД-вызовов.

### Цель

Написать приложение, которое использует Virtual Threads и измеряет время выполнения всех задач.

### Дополнительно для эксперимента

- Попробовать заменить Virtual Thread Executor на обычный `FixedThreadPool` (например, 100 потоков).
- Сравнить время выполнения и поведение программы.
- Сделать выводы, почему Virtual Threads масштабируются лучше.
